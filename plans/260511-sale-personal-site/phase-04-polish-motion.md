# Phase 04 Polish And Motion

## Overview

Add the visual layer that makes the site memorable without reducing clarity.

## Requirements

- background feels atmospheric
- motion is subtle and meaningful
- hover states improve feedback
- page still performs well

## Architecture

Polish layer should stay mostly in CSS.
Only add JS-driven animation if the benefit is obvious.

## Implementation Steps

1. Add gradient and glow composition to background.
2. Add subtle texture or noise if needed.
3. Add entrance animations for sections.
4. Add stagger reveal for product cards.
5. Add hover transitions for cards and buttons.
6. Review contrast and reduce excess effects.

## Acceptance Criteria

- no distracting animation loops
- visual polish improves first impression
- content readability stays high
- mobile performance remains acceptable

## Risks

- animation overuse
- too many layered blurs
- expensive paint effects on low-end devices
